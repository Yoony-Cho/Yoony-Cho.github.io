## HTML과 XML?

**HTML**(HyperText Markup Language)과 **XML**(Extensible Markup Language)은 모두 마크업 언어로, 웹사이트나 데이터베이스에서 데이터를 표현하기 위한 언어입니다.

#### HTML

HTML은 대표적인 마크업 언어이며, 웹 브라우저에서 화면에 표현됩니다. HTML은 컴퓨터의 코드를 사용자가 이해할 수 있는 형태로 변환하여 표현합니다. 주로 웹 페이지를 만들 때 사용되며, 문서의 내용과 구조를 표현합니다.

#### XML

XML은 **데이터를 표현하기 위한 마크업 언어**입니다. 다양한 시스템, 언어 간의 연동 및 데이터 전송의 모든 목적에 사용됩니다. XML은 데이터를 저장하는 데에 공통적으로 사용됩니다.



### HTML과 XML의 공통점과 차이

HTML과 XML은 모두 마크업언어로서, 문서나 데이터의 구조를 태그로 정의합니다. 그러나 두 언어 사이에는 중요한 차이점이 있습니다.

공통점:

1. 마크업언어로서 구조화된 정적 문서를 만드는 데에 사용됩니다.
2. 태그를 사용하여 데이터를 구조화합니다.
3. 웹 페이지에서 사용되는 모든 문서는 HTML와 XML의 태그를 사용해 작성됩니다.

차이점:

1. 용도

- HTML은 웹 페이지의 내용 및 표현과 관련된 내용을 작성할 때 사용됩니다.
- XML은 데이터의 구조와 의미를 정의합니다.

2. 문법

- HTML은 태그를 갖지만, XML에서는 사용자가 직접 태그를 정의할 수 있습니다.
- HTML에서는 구조를 만들기 위해 특별한 요소와 특별한 규칙이 있습니다.
- XML에서는 개발자 또는 사용자가 데이터의 구조 및 형식을 직접 정할 수 있습니다.

3. 규칙

- HTML에서는 태그를 사용할 때, 열린 태그와 닫힌 태그, 인용부호, 대소문자 구분 등 언어 규칙을 준수해야 합니다.
- XML에서는 문서의 규범적 규칙을 따릅니다. 데이터를 정의하기 위해 개발자나 사용자가 직접 태그를 정할 수 있으며, 시작 태그와 마침 태그를 꼭 써줘야합니다.

4. 확장성

- HTML은 복잡한 데이터 구조를 다루지 못하므로 확장성이 제한되어 있습니다.
- XML은 데이터의 필요에 따라 태그를 추가하거나, 정의를 바꿀 수 있기 때문에 더 많은 데이터를 구조화할 수 있습니다.

5. 사용

- HTML은 웹 페이지를 만들 때 사용됩니다.
- XML은 데이터의 전송, 저장, 검색 등에 사용됩니다.
